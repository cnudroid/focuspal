## Priority & Severity Guidelines

### Priority Levels
- **P0 (Critical)**: Blocks release, fixes app crashes, data loss, or security issues
- **P1 (High)**: Major feature broken, but workaround exists
- **P2 (Medium)**: Minor feature issue, cosmetic bugs
- **P3 (Low)**: Nice-to-have improvements, edge cases

### Severity Levels
- **Critical**: App crashes, data loss, security vulnerability
- **High**: Major feature completely broken, incorrect data
- **Medium**: Feature works but has issues, minor data inconsistencies
- **Low**: Cosmetic issues, minor inconvenience
